::: tip
本文对应参考文档中的“实例管理”和“任务列表”说明，适合刚开始整理自己的预设时查阅。
:::

## 一、实例管理

### 新建与切换

- 在界面最上方操作。
- 点击即可切换当前实例。

### 右键实例可做的事

- 复制
- 导出配置
- 导入配置
- 重命名

## 二、任务列表常用操作

| 操作 | 说明 |
| --- | --- |
| 全选 / 全不选 | 一键勾选或取消所有任务 |
| 添加任务 | 预置任务不足时手动补充，也可查找目标任务 |
| 重置任务 | 恢复为内置预设，会覆盖当前预设内容 |
| 修改任务名 | 在任务名旁空白处右键 |
| 单独运行 / 从此处运行 | 便于局部调试任务流程 |
| 删除任务 | 直接从当前实例移除 |
| 跳过任务 | 右键任务名后，当前轮运行会跳过该任务 |
| 任务排序 | 拖动任务即可调整顺序 |

## 三、任务列表的使用建议

1. 日常任务适合每次运行执行一次。
2. 需要持续刷取的内容，优先使用特色功能或循环任务。
3. 改动较多时，先复制实例再试，不要直接在主用实例上大改。

## 四、多开与总控台

`V2.1.0` 之后支持 `1VN` 个模拟器，旧版本通常需要复制多个本体做 `1V1` 操作。

### 多开基础要求

- 电脑性能要足够
- 需要了解基础 `ADB` 使用
- 模拟器本身要支持多开
- 建议善用搜索与本手册排查端口问题

### 创建新实例并连接

1. 点击主页右上角 `+` 新建实例。
2. 点击连接框的 `🔄` 刷新按钮自动检测端口。
3. 或点击“自定义”手动填写 `ADB` 路径和地址。
4. 打开“记住连接”，让实例记住当前模拟器。

### 总控台说明

- 入口在左侧边栏。
- 可查看各实例实时画面与连接状态。
- 支持启动 / 停止任务。
- 总控台偏监控和控制用途，新建或编辑连接仍在主界面完成。

## 五、启动配置

如果需要自动启动模拟器：

1. 打开“启动后操作 → 启动模拟器并开始任务”。
2. 在“设置 → 启动设置”填写模拟器路径和附加命令。
3. 多开号参数可通过快捷方式目标栏或任务管理器命令行查看。

::: details 启动参数示例
- MuMu 常见参数形如 `-v 0`
- BlueStacks 常见参数形如 `--instance Nougat32`
:::
