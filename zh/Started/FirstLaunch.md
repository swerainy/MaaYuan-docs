# 首次启动<Badge type="warning" text="普通" />

::: tip
本文说明首次打开 MaaYuan 时的依赖检测、启动方式、权限设置与模拟器基础要求。
:::

## 一、第一次启动会发生什么

1. MaaYuan 会先尝试自动安装或检测依赖库。
2. 首次启动时等待 10~15 秒属于正常现象。
3. 之后每次启动前先做依赖检测，也属于正常行为。

## 二、Windows 启动方法

1. 双击运行 `MaaYuan.exe`。
2. 如果首次运行报错，可在安装目录找到 `install-deps.ps1`。
3. 右键选择“使用 PowerShell 运行”，完成后重启电脑。

## 三、macOS 权限设置

1. 打开终端。
2. 执行以下命令：

```sh
sudo spctl --master-disable
```

3. 输入系统密码并回车。
4. 打开“系统设置 → 隐私与安全性 → 安全性”，选择“任何来源”。
5. 若底部出现“已阻止使用的应用”，点击“仍然允许”。
6. 如果多次弹窗拦截，保持该设置页打开，重复点击“仍然允许”。

::: warning
如果双击 MaaYuan 后没有弹出主窗口，请重新执行权限命令，并再次检查“隐私与安全性”页面。
:::

## 四、模拟器要求

| 项目 | 推荐值 |
| --- | --- |
| 分辨率 | `720 × 1280` |
| DPI | `320` |
| 比例 | `16:9` 或 `9:16` |

建议在开始使用前就把模拟器分辨率固定下来，避免后续识图偏差。

## 五、第一次开始任务

1. 打开 MaaYuan。
2. 勾选你想执行的任务。
3. 点击“开始任务”。
4. 如果需要理解各任务用途，继续阅读 [任务总览](../Manual/TaskOverview) 或 [功能介绍](../Features/InstancesAndTaskList)。

## 六、常见首次启动异常

- 长时间卡在依赖安装：去看 [FAQ](../FAQ/)
- 双击无反应：检查运行库与系统权限
- 模拟器未适配：优先确认分辨率、ADB 连接与后台保活设置
