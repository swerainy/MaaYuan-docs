# 连接与更新<Badge type="warning" text="普通" />

::: tip
本文说明首次连接模拟器、自动/手动连接方法，以及更新渠道与版本选择。
:::

## 一、自动连接模拟器

1. 先自行安装并启动模拟器。
2. 打开游戏并停留在能看到头像与昵称的页面。
3. 打开 `MaaYuan` 。
4. 在右上角连接框点击右起第二个 `🔄` 刷新按钮。
5. MaaYuan 会自动检测当前运行的单个模拟器，并填充 `ADB` 路径、连接地址和配置。

## 二、手动连接模拟器

如果自动检测失败，可以改为手动填写：

1. 点击连接框中的“自定义”。
2. 填写 `ADB` 路径。
3. 填写连接地址，例如 `127.0.0.1:<端口号>` 或 `emulator-<四位数字>`。
4. 点击保存。

::: details 常见模拟器地址示例
- `BlueStacks 5`：`127.0.0.1:5555/5556/5565/5575/5585/5595/5554`
- `MuMu 12`：`127.0.0.1:16384/16416/16448/16480/16512/16544/16576`
- `雷电 9`：`emulator-5554/5556/5558/5560` 或 `127.0.0.1:5555/5557/5559/5561`
- `夜神`：`127.0.0.1:62001/59865`
- `逍遥`：`127.0.0.1:21503`
:::

## 三、建议的连接设置

- 打开“记住连接”，下次打开该实例会默认连接当前模拟器。
- 若使用自定义连接，不要再点“刷新”或“自动检测”，避免把路径覆盖回错误值。
- 若目标模拟器本身以管理员权限运行，MaaYuan 有时也需要管理员权限。

## 四、更新设置

1. 打开 `MaaYuan` 的“更新设置”。
2. 优先推荐选择“正式版”。
3. 更新渠道可按需选择 `Mirror酱` 或 `GitHub`。

### 版本区别

- **正式版**：适合仅需稳定日常使用的用户。
- **公测版**：含当前活动适配新任务，适合有一定电脑基础且能自主排查BUG的用户。
- **内测版**：**不要选**（不主动发布），选了不负责维护。

## 五、更新方式

### 内置更新

- `GitHub`：通常需要科学上网。
- `Mirror酱`：国内更省事，适合自动更新。

### 外部手动更新

1. 完整包更新：删除旧目录内容后，重新解压完整包。
2. `update` 包更新：按更新说明删除指定文件后覆盖替换。

::: warning
更新前如需保留配置，请先导出实例或配置文件；更新失败时不要反复覆盖未知状态的旧目录，优先直接下载完整包重新解压。
:::
